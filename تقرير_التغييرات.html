📁 تفاصيل التغييرات لكل ملف

main.py — الملف الرئيسيتعديل
نظام تسجيل الدخول التلقائي: أُضيفت خيارات --login-url و --username و --password. الأداة تفتح صفحة الدخول، تستخرج حقول النموذج، تُرسل البيانات، وتحفظ session cookie للاستخدام في الفحص.
دمج وحدة فحص رفع الملفات: أُضيف UploadScanner كوحدة جديدة مع خيار --upload.
إزالة التكرارات: دالة deduplicate_vulnerabilities() تزيل الثغرات المتكررة تلقائياً قبل التقرير.
فحص متتابع: تغيير Phase 3 من threads إلى sequential لحماية cookies الجلسة.
توقيت المصادقة: نقل auto-login إلى ما بعد Phase 1 لتجنب خطأ SQLite database is locked.
تتبع النماذج: إضافة set لتتبع النماذج المفحوصة وتجنب فحص نفس النموذج مراراً.

modules/crawler.py — زاحف الويبتعديل
تتبع إعادة التوجيه: يتابع 301/302 ويُضيف الوجهة النهائية.
اكتشاف form actions: يستخرج روابط النماذج كمسارات إضافية.
استخراج URLs من JavaScript: يبحث عن روابط مدمجة في كود JS.
زيادة العمق: من max_depth=2 إلى max_depth=3.
modules/form_scan.py — فحص النماذجتعديل
كشف CSRF المفقود: يفحص كل نموذج POST بحثاً عن tokens الحماية (Django, Laravel, Rails, .NET).
كشف حقول رفع الملفات: يحدد النماذج التي تحتوي input type="file".
دعم عناصر إضافية: textarea و select بالإضافة لـ input.
modules/header_scan.py — فحص الأمانتعديل
فحص أمان Cookies: يتحقق من وجود HttpOnly و Secure flags على جميع cookies الاستجابة.
modules/sqli_scan.py — فحص SQL Injectionتعديل
كشف Login Bypass: يقارن استجابة SQLi مع استجابة عادية — إذا SQLi أعطى 302 والعادي أعطى 200 = Login Bypass!
Boolean-Blind SQLi: يُرسل AND 1=1 (TRUE) و AND 1=2 (FALSE) ويقارن حجم الاستجابة.
أخطاء Django/Python: إضافة operationalerror, django.db.utils, psycopg2, pymysql, sqlite3.
payloads جديدة: Union Select و Error-based payloads.
modules/lfi_scan.py — فحص تضمين الملفات المحليةتعديل
تحليل روابط الصفحة: يبحث عن جميع الروابط التي تحتوي parameters مثل file, page, doc, path, include, template, view.
payloads متنوعة: path traversal, absolute path, double encoding, أنظمة Linux و Windows.
modules/upload_scan.py — فحص رفع الملفاتملف جديد
فحص رفع ملفات خطيرة: يبحث عن نماذج الرفع ويحاول رفع:
• test_scan.php — محاكاة PHP web shell
• test_scan.exe — ملف تنفيذي
• test_scan.html — ملف HTML مع JavaScript
ملء النماذج تلقائياً: يملأ الحقول المخفية والنصية بقيم آمنة.
utils/reporter.py — التقاريرتعديل
تصنيفات OWASP جديدة: CSRF → A01:2021, Cookie → A05:2021, Upload → A04:2021.
نصائح إصلاح: توصيات لكل نوع ثغرة جديد.

🛠️ دليل حل المشاكل

❌ المشكلة: Auto-login failed
السبب: قاعدة بيانات SQLite مقفلة من اتصالات الفحص السابقة.
الحل:
kill $(lsof -i :8000 -t) 2>/dev/null
sleep 2
cd /Users/mohadreamer/AbaNeu
source venv/bin/activate
python3 manage.py runserver 8000 &
sleep 3
# ثم أعد الفحص
❌ المشكلة: Connection refused
السبب: السيرفر غير مشغّل.
الحل:
cd /Users/mohadreamer/AbaNeu && source venv/bin/activate
python3 manage.py runserver 8000 &
❌ المشكلة: That port is already in use
السبب: عملية أخرى تستخدم البورت 8000.
الحل:
kill -9 $(lsof -i :8000 -t)
sleep 2
python3 manage.py runserver 8000 &
❌ المشكلة: لم يُكتشف سوى Headers و Directories
السبب: فشل تسجيل الدخول فالأداة لم تصل للصفحات المحمية.
الحل: أعد تشغيل السيرفر (انظر المشكلة الأولى) ثم أعد الفحص.
